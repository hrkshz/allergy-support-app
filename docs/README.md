# Docs Guide

この `docs/` には、プロダクト全体像と判定ロジックの説明資料を置いています。

## 資料一覧

[`portfolio-system-overview.md`](./portfolio-system-overview.md)

- このプロダクトが何を解くのか
- どういう流れで使うのか
- なぜこの構成を選ぶのか
- 構成・データ配置・API層をどう考えるのか
- フロント配信を `Vercel`、バックエンドを `Supabase` でどう分けるのか

をまとめた、全体像の説明資料です。  
プロダクトの構成や体験の流れを確認したいときに見ます。

[`allergen-detection-logic.md`](./allergen-detection-logic.md)

- 判定がどう組み立てられるのか
- `検出なし / 要確認 / 検出あり` をどう考えるのか

をまとめた、判定ロジックの補助資料です。  
結果の作り方や判定の考え方だけを確認したいときに見ます。

[`system-components-catalog.md`](./system-components-catalog.md)

- 登場するコンポーネントを細かい粒度で一覧できる
- `サーバ / DB / 外部API / 画面 / ルール` の違いを見分けやすい
- 各要素がこのシステムで何をしているかを初心者向けに確認できる

をまとめた、構成要素の索引資料です。  
「この部品は何者か」「どの種類に属するのか」を整理して把握したいときに見ます。

[`system-request-flow.md`](./system-request-flow.md)

- Web アプリにアクセスしてから最初の画面が出るまでの流れを追える
- フロントコードがどこに保管され、どう配信され、どこで動くかを確認できる
- ログイン、画像保存、Edge Functions、OpenAI、DB保存までの時系列を確認できる

をまとめた、システム通信と実行の流れの補助資料です。  
「どのコンポーネントが、いつ、どこで動くか」を時系列で整理したいときに見ます。
